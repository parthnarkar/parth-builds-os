## üöÄ Getting Started

### Step 1: Fork the Repository

Click the "Fork" button at the top right of the repository page.

### Step 2: Clone Your Fork

```bash
git clone https://github.com/parthnarkar/parth-builds-os.git
cd parth-builds-os
```

### Step 3: Create a Branch

```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names:
- `feature/add-tree-problems`
- `fix/linked-list-typo`
- `docs/improve-array-explanation`

### Step 4: Make Your Changes

Follow the [contribution guidelines](#contribution-guidelines) below.

### Step 5: Commit Your Changes

```bash
git add .
git commit -m "Add: Binary Tree traversal problems"
```

See [commit message guidelines](#commit-messages) below.

### Step 6: Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### Step 7: Open a Pull Request

Go to the original repository and click "New Pull Request". Provide a clear description of your changes.

## ‚úÖ Contribution Guidelines

### For DSA Content

When adding a problem, include:

```markdown
## Problem Title

**Difficulty:** Easy/Medium/Hard  
**Problem Link:** [LeetCode/GeeksforGeeks link]

### Problem Statement
Clear description of the problem with examples

### Approach
Intuitive explanation of the solution approach

### Code Implementation
```java
// Well-commented code
```

### Complexity Analysis
- **Time Complexity:** O(n)
- **Space Complexity:** O(1)

### Edge Cases
- Empty input
- Single element
- Large inputs
```

### For Documentation

- Use clear, concise language
- Include examples where helpful
- Check for spelling and grammar
- Maintain consistent formatting

### Quality Standards

- **Code Quality:** Clean, readable, well-commented code
- **Completeness:** Include all required sections
- **Testing:** Verify your code works correctly
- **Originality:** Share your unique perspective

## üé® Style Guide

### File Naming

- Use descriptive names: `Binary-Search-Trees.md`
- Use hyphens for spaces
- Capitalize major words

### Code Formatting

**Java:**
```java
public class Solution {
    public int exampleMethod(int[] nums) {
        // Clear comments
        int result = 0;
        return result;
    }
}
```

**JavaScript:**
```javascript
function exampleFunction(arr) {
    // Clear comments
    let result = 0;
    return result;
}
```

### Markdown Formatting

- Use `##` for main headings
- Use `###` for subheadings
- Use code blocks with language specification
- Use bullet points for lists
- Include blank lines between sections

## üí¨ Commit Messages

Use clear, descriptive commit messages:

### Format
```
Type: Brief description

Optional detailed description
```

### Types
- `Add:` New content or features
- `Fix:` Bug fixes or corrections
- `Update:` Improvements to existing content
- `Docs:` Documentation changes
- `Style:` Formatting changes

### Examples
```bash
git commit -m "Add: Merge Sort implementation with explanation"
git commit -m "Fix: Typo in linked list Set 2"
git commit -m "Update: Improve time complexity explanation"
git commit -m "Docs: Add contributing guidelines"
```

## üîç Review Process

1. **Automated Checks:** Basic formatting and file structure
2. **Manual Review:** Code quality, explanation clarity
3. **Feedback:** Constructive suggestions for improvement
4. **Approval:** Merge when all criteria are met
